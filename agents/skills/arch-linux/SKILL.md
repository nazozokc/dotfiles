# Arch Linux ファイル操作ガイドライン

## 触れてはいけない領域（絶対禁止）
- `/etc/` 配下のシステム設定
- `/usr/` 配下のシステムファイル
- `/var/` 配下の変数データ
- `sudo` が必要な全ての操作
- システムパッケージの 直接編集

##  操作して良い領域
- `~/.config/` 配下のユーザ設定
- `~/` (ホームディレクトリ)内の個人用ファイル
- このリポジトリ（dotfiles）内のファイル

## 操作時の心得
- **指示以外はやらない**: ユーザーが明確に依頼した事のみ実行
- **「、ついでに」は禁止**: 関連作業を勝手に行わない
- **確認を取る**: 危険そうな操作は事前に報告
- **バックアップ**: 重要な変更前は現在の状態を保存

## 重要な判断基準
「これはシステム設定か？」 → 自信がなければ触らない
「これは個人設定か？」 → dotfiles 内ならOK
